1、什么是平台总线模型？
	平台总想模型也叫platform总线模型。是LLinux内核虚拟出来的一条总线，不是真实的导线。

2、为什么会有平台总线模型？
	1）可以提高代码的重用性
	2）减少重复性代码
	原因：
	没有总线时，设备和驱动写在同一个c文件
	总线将设备和驱动分开，写在两个c文件
	如，device.c内部写着和硬件有关的（如引脚和寄存器等需要变的代码）
		driver.c内部写着和硬件无关的代码（如逻辑控制等稳定不变的代码）

3、平台总线模型的优点
	1）减少代码重复性
	2）便于管理

4、怎么编写以平台总线模型设计的驱动？
	一个device.c，一个driver.c，然后分别注册device.c和driver.c
	在insmod时，平台总线以名字来匹配device.c和driver.c，即字符串比较。
